# 项目大纲

> 最后更新: 2026-05-23

## 一、项目定位

基于 [pi coding agent](https://github.com/badlogic/pi-mono) 的**通用角色扮演引擎**。不绑定特定世界观——导入任意 SillyTavern 角色卡即可扮演。

## 二、核心特性

| 特性 | 状态 |
|------|------|
| 多卡片并发（跨世界融合） | ✅ |
| SillyTavern PNG/JSON 角色卡导入 | ✅ |
| 导入时自动预处理（正则→钩子、酒馆→变量、URL扫描） | ✅ |
| 卡片完全隔离（状态、世界书、正则、session） | ✅ |
| 变量系统（Schema 校验、值域钳制、事件追踪） | ✅ |
| AI 本地化接口（ui/、scripts/、vectors/） | ✅ |
| 正则脚本全链路（引擎剥离 + 前端渲染） | ✅ |
| 前端卡片 UI 动态加载 | ✅ |
| 世界书主动注入（Token 预算 1500） | ✅ |
| 用户轮数计数（基于 turn_end，不受 steer 污染） | ✅ |
| RP Web 前端（消息渲染、状态面板、卡片管理） | ✅ |
| 动态关键词索引（替代硬编码角色名列表） | ✅ |
| 向量检索 | 📋 待办（POST_IMPORT_TODO.md） |
| MVU json_patch 本地化 | 📋 待办 |

## 三、数据流

```
角色卡 PNG
  │
  ▼
setup.mjs（14 步自动流程）
  ├── 解析 → 提取 character_book / regex_scripts / tavern_scripts
  ├── 预处理 → regex_hooks.json / variable_schema.json
  ├── 扫描 → remote_urls.json
  ├── 生成 → state.json / config.json / APPEND_SYSTEM.md
  ├── 注册 → registry.json
  └── 标记 → POST_IMPORT_TODO.md
  │
  ▼
引擎启动（session_start）
  ├── 加载卡片状态（cardStates 格式）
  ├── 编译正则钩子（prompt 剥离 + display 替换）
  ├── 加载变量 Schema（按 cardId 隔离）
  ├── 建立动态关键词索引（遍历世界书文件）
  ├── 设置卡片独立 sessionDir
  └── 下发 UI 组件（ui/ 目录）→ WebSocket → 前端
  │
  ▼
对话循环
  ├── turn_start：世界书主动注入
  ├── AI 调用工具：read_state / update_state / load_worldbook
  ├── message_end：应用正则钩子
  └── turn_end：用户轮数 +1，写回卡片 state.json
  │
  ▼
前端渲染管道
  ├── 解析 XML → content / choice / perspective
  ├── Markdown → HTML
  ├── 执行 display 正则钩子
  ├── 渲染状态栏（卡片 UI 组件）
  └── 注入 DOM
```
