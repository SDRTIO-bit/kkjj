# 🔧 从零复现

> **目标**：空目录开始搭一个能跑的 pi 角色扮演系统。
> 这里只记录步骤和踩过的坑。

## 前置条件

```bash
pi --version    # ≥ v0.75
node --version  # ≥ 18
```

## 步骤总览

```
① 创建目录结构
② 配置 settings.json
③ 编写角色卡（角色设定 + 角色初始状态）
④ 编写世界书
⑤ 创建 setup.mjs 初始化脚本
⑥ 编写 pi 扩展 rp-engine.ts
⑦ 编写前端页面 rp-web/
⑧ 配置 APPEND_SYSTEM.md
⑨ 启动运行
```

---

## 一、创建目录结构

```bash
mkdir -p .pi/extensions/rp-web
mkdir -p .pi/skills
mkdir -p .pi/prompts
mkdir -p .pi/worldbook/世界观
mkdir -p .pi/worldbook/角色设定
mkdir -p .pi/worldbook/角色初始状态
mkdir -p .pi/worldbook/身体演化
mkdir -p .pi/worldbook/格式指令
```

## 二、配置文件

**`.pi/settings.json`**
```json
{
  "skills": ["./skills/rp/SKILL.md"],
  "enableSkillCommands": true,
  "sessionDir": ".pi/sessions"
}
```

> ⚠️ **sessionDir 是坑**：不配的话对话记录存在 `~/.pi/agent/sessions/` 全局目录，和普通 pi 对话混一起。配了 `.pi/sessions` 就存项目本地。

## 三、角色卡

角色卡分两种文件，放不同目录：

### 角色设定（给 AI 读的文字描述）

`.pi/worldbook/角色设定/角色名.md`

```markdown
# 角色名
## 基本信息
- 姓名: xxx
- 年龄: xx
- 身份: xxx

## 外貌
...

## 性格
...
```

### 角色初始状态（游戏数据，被 setup.mjs 解析）

`.pi/worldbook/角色初始状态/角色名.md`

```markdown
# 角色名 - 初始状态

```yaml
  归属值: 0
  情分值: 100
  基本信息:
    姓名: 角色名
  # 其他字段按需加
```
```

一个角色对应两个文件。

## 四、世界书

世界书是 AI 的知识库，通过 `load_worldbook` 工具按关键词搜索。

文件放 `.pi/worldbook/世界观/` 下，命名建议：

```
世界观 花.md              ← 分类前缀 + 空格 + 标题
世界观纯爱线 花.md        ← 路线专属用不同前缀
```

> ⚠️ 文件名不能有连续多个空格。`setup.mjs` 的 `cleanName` 函数会自动清理，但源文件命名时注意。

文件里第一行建议写关键词：
```markdown
# 世界观 花
> 关键词: 花, 开花, 花朵

内容...
```

## 五、初始化脚本 setup.mjs

这个脚本干两件事：

1. 把 `worldbook_clean/` 下的世界书文件复制到 `.pi/worldbook/`（清理文件名空格）
2. 从 `角色初始状态/` 中读取 YAML 数据块，解析后生成 `.pi/state.json`

运行方式：

```bash
node setup.mjs
```

每次改角色初始状态文件后都要重新运行。

## 六、pi 扩展 rp-engine.ts

扩展是 pi 的插件机制。这个文件干了这些事：

| 模块 | 作用 |
|------|------|
| 状态管理 | 读写 state.json，记录变更历史 |
| 世界书搜索 | load_worldbook 工具的实现 |
| 4 个工具 | read_state / update_state / advance_time / load_worldbook |
| 系统提示注入 | 把角色状态概要注入到 AI 的 prompt 中 |
| Web 服务器 | 启动一个 HTTP + WebSocket 服务 |

### 常见报错

<details>
<summary><strong>报错 1：</strong> <code>Failed to load extension "xxx.js": window is not defined</code></summary>

**原因**：pi 扫描 `extensions/` 目录，把前端 JS 文件也当扩展加载了。
**解决**：前端文件放 `extensions/rp-web/` 子目录，pi 不递归扫描子目录。
</details>

<details>
<summary><strong>报错 2：</strong> <code>Cannot find module 'ws'</code></summary>

**原因**：WebSocket 库没装。
**解决**：`npm install ws`
</details>

<details>
<summary><strong>报错 3：</strong> <code>latestCtx is not defined</code></summary>

**原因**：变量在事件处理函数中引用时还没赋值。
**解决**：把声明移到函数外部作为模块级变量。
</details>

<details>
<summary><strong>报错 4：</strong> RP Web 页面返回 <code>Not Found</code></summary>

**原因**：`path.join` 中路径以 `/` 开头，被当作绝对路径。
**解决**：去掉路径开头的 `/`。
</details>

<details>
<summary><strong>报错 5：</strong> 两个 <code>session_start</code> 事件冲突</summary>

**原因**：多个扩展注册同一事件，后者被覆盖。
**解决**：合并到同一个 `session_start` 里。
</details>

## 七、前端页面 rp-web/

前端文件放在 `extensions/rp-web/` 子目录，作为静态资源被 Web 服务器加载。

| 文件 | 作用 |
|------|------|
| `rp-web.html` | 页面结构，RP 开关、状态面板、会话选择器 |
| `rp-web-app.js` | 主逻辑：WebSocket 连接、消息处理 |
| `rp-web-message-renderer.js` | 消息渲染：XML 标签 → HTML |
| `rp-web-style.css` | 样式 |

### 消息渲染流程

```
AI 输出（含 XML 标签）
  → renderAssistantMessage()
    → _processRPContent()    // RP 模式开关
      → 开: _renderWithPerspectives()
        → 解析 <perspective> → 可折叠卡片
        → 解析 <choice> → 可点击按钮
        → 去掉 <thinking> <UpdateVariable>
      → 关: _stripXMLTags()
        → 去掉所有 XML 标签，只留纯文本
    → renderMarkdown()       // markdown 转 HTML
```

### 为什么前端做正则替换？

后端（pi 进程）只透传 AI 输出，不改内容。前端处理的好处：
- 不改原始消息，可回看原始格式
- RP 模式开关控制是否生效
- 纯前端逻辑，不增加后端负担

## 八、APPEND_SYSTEM.md

这个文件 pi 每次对话都会自动注入到系统提示，用来控制 AI 的输出风格和格式。

```markdown
# 风格规范
- 叙述风格：第三人称，细腻描写
- 标签格式：<thinking> <content> <perspective> <choice> <UpdateVariable>
- 所有标签必须完整闭合
```

> ⚠️ `APPEND_SYSTEM.md` 是全局注入的，不 RP 时也会生效。

## 九、启动

```bash
node setup.mjs    # 初始化
pi                # 启动
```

在 pi 中 `/route 纯爱线` 选择路线。浏览器打开 `http://localhost:3012`。

## 踩坑速查表

| 症状 | 原因 | 解决 |
|------|------|------|
| `window is not defined` | pi 把前端 JS 当扩展加载 | 前端文件放 `rp-web/` 子目录 |
| `Not Found` | path.join 路径以 `/` 开头 | 去掉开头的 `/` |
| 对话历史混在一起 | 没配 sessionDir | settings.json 加 `sessionDir` |
| 改角色状态不生效 | 忘记运行 setup.mjs | `node setup.mjs` |
| 两个扩展抢事件 | session_start 冲突 | 合并事件处理 |
