# 脚本与正则改造指南

> 本文档说明如何将 SillyTavern 角色卡中的正则脚本/酒馆脚本，改造为 RP Engine 可用的功能。

---

## 一、总览：正则脚本处理全链路

```
角色卡导入 (setup.mjs)
  │
  ├── 原始脚本 → regex_scripts/*.json（原样备份）
  │
  └── ⭐ 预处理 → 分类筛选，生成 regex_hooks.json（见第二节）
        │
        ▼
引擎启动 (rp-engine/index.ts)
  │
  ├── session_start: loadRegexHooks()
  │    读取所有激活卡片的 regex_scripts/*.json
  │    编译正则 → 按 promptOnly 分两类：
  │
  │    ├── prompt 钩子：剥离型（去除变量更新、隐藏AI内容等）
  │    │     在 message_end 时执行 → 内容不在上下文中留下
  │    │
  │    └── display 钩子：替换型（SFW_IMG → img 等）
  │          在 message_end 时执行 → 修改后的内容下发前端
  │
  └── rp-web-server: WebSocket 连接时下发 display 钩子给前端
        │
        ▼
前端 (rp-web-message-renderer.js)
  │
  └── _processRPContent() 渲染管道：
        extract XML → Markdown→HTML → ⭐ 执行 display 钩子 → 注入 DOM
```

---

## 二、自动预处理（setup.mjs 导入时）

从 v3 开始，`node setup.mjs --import <角色卡>` 会在导入后自动执行以下预处理：

### 2.1 正则脚本 → 渲染钩子 (`regex_hooks.json`)

| 原格式 | 预处理结果 |
|--------|-----------|
| SillyTavern `findRegex` + `replaceString` | 转为 JS RegExp pattern + flags + HTML template |
| `placement: [1,2]` | 标记为 display 阶段（Markdown→HTML 之后） |
| `promptOnly: true` | 跳过前端渲染，保留在引擎层 prompt 阶段剥离 |
| `disabled: true` | 跳过 |
| 含「状态栏」「选项」「开场」的脚本 | 跳过（依赖外部远程资源，无法在 Web 前端运行） |
| 含「对AI隐藏」的脚本 | 跳过（引擎层直接处理原始 regex_scripts） |

处理后产物：`.pi/cards/<卡名>/regex_hooks.json`

### 2.2 酒馆脚本 → 角色变量定义 (`variable_schema.json`)

从「变量结构设计」脚本中提取 Zod Schema，解析为角色字段定义，自动合并到 `state.json`。详见 [#角色变量预处理](#四角色变量预处理)。

## 三、引擎层：正则钩子编译与应用

### 3.1 模块：`regex-processor.ts`

位置：`.pi/extensions/rp-engine/regex-processor.ts`

核心函数：

| 函数 | 用途 |
|------|------|
| `loadRegexHooks(cardDirs)` | 从所有激活卡片加载原始正则脚本，编译为 `prompt[]` + `display[]` 两类钩子 |
| `compileSTRegex(findRegex)` | 将 SillyTavern 的 `/pattern/flags` 格式转为 JavaScript `RegExp` |
| `applyPromptHooks(text, hooks)` | 对文本执行所有 prompt 阶段钩子（剥离隐藏内容） |
| `applyDisplayHooks(text, hooks)` | 对文本执行所有 display 阶段钩子（内容替换） |

### 3.2 加载时机：`session_start`

```typescript
// index.ts - session_start 事件
const activeCardsList = getActiveCards();
const cardDirs = activeCardsList.map((c) => c.dir);
const hooks = loadRegexHooks(cardDirs);
compiledHooks = [...hooks.prompt, ...hooks.display];
```

### 3.3 应用时机：`message_end`

每次 AI 助理消息生成完毕后，在 `message_end` 事件中对消息内容应用钩子：

```typescript
// index.ts - message_end 事件
pi.on("message_end", async (event: any) => {
  if (event?.message) {
    applyHooksToMessage(event.message);
  }
});

function applyHooksToMessage(msg: any): void {
  if (!msg || msg.role !== "assistant") return;
  // 1. prompt 钩子：剥离不应保留的内容
  msg.content = applyPromptHooks(msg.content, compiledHooks);
  // 2. display 钩子：替换显示内容
  msg.content = applyDisplayHooks(msg.content, compiledHooks);
}
```

**为什么在 message_end 执行？**
- prompt 钩子剥离的内容不会进入对话上下文（防止 AI 循环看到已剥离的标签）
- display 钩子替换后的内容直接发给前端渲染

### 3.4 前端下发：WebSocket 连接时

```typescript
// rp-web-server.ts - WebSocket connection
if (getRegexHooks) {
  const hooks = getRegexHooks();
  if (hooks.display.length > 0) {
    sendToRP(ws, { type: "regex_hooks", hooks: hooks.display });
  }
}
```

## 四、前端层：display 钩子执行

### 4.1 接收钩子（`rp-web-app.js`）

```javascript
case 'regex_hooks':
  if (msg.hooks && Array.isArray(msg.hooks)) {
    messageRenderer.setRegexHooks(msg.hooks);
  }
  break;
```

### 4.2 应用钩子（`rp-web-message-renderer.js`）

在 `_processRPContent()` 的渲染管道中，**Markdown→HTML 之后、DOM 注入之前**执行：

```javascript
// 渲染正文
let result = renderMarkdown(parsed.mainContent);

// ⭐ 应用 display 阶段正则钩子
if (this.regexHooks.length > 0) {
  for (const hook of this.regexHooks) {
    const regex = new RegExp(hook.pattern, hook.flags);
    result = result.replace(regex, hook.replacement);
  }
}
```

**为什么在 Markdown→HTML 之后？**
- `<SFW_IMG>` 等标签不是标准 Markdown
- 替换为 `<img>` 标签后，最终注入 DOM 时已经是完整的 HTML

## 五、角色变量预处理

### 5.1 预处理流程

```
酒馆脚本「变量结构设计」(Zod Schema)
  │
  ▼
setup.mjs: preprocessTavernScripts()
  │
  ├── 解析 z.object({...}) → 提取角色名和字段
  ├── 写入 variable_schema.json（变量定义快照）
  └── 合并到 state.json（角色字段 + 事件标记）
```

### 5.2 变量更新

| 原 SillyTavern | 改造后 RP Engine |
|---------------|-----------------|
| MVU Zod 脚本拦截 `_.set()` | AI 调用 `update_state` 工具 |
| 变量存储在世界书条目中 | 变量存储在 `state.json` 角色字段中 |
| 每次对话开始时从世界书加载 | 每次 session_start 从 state.json 加载 |

AI 通过 `read_state` 可查看所有角色变量（包括预处理合并的自定义字段），通过 `update_state` 更新任意字段。归属值更新时情分值自动同步（`情分值 = 100 - 归属值`）。

## 六、跨卡片隔离

| 隔离项 | 机制 |
|--------|------|
| 正则脚本 | 每张卡独立的 `regex_scripts/` 目录，引擎按激活卡片加载，`loadRegexHooks` 中标记来源 `cardId` |
| 渲染钩子 | 每张卡独立的 `regex_hooks.json`，前端按 WebSocket 下发的钩子列表执行 |
| 角色状态 | 每张卡独立的 `state.json`，运行时合并到全局 `cardStates` |
| 变量 Schema | 每张卡独立的 `variable_schema.json`，不同卡的 `背德值`、`欲望值` 等互不干扰 |

不同卡片的同名角色不会冲突——它们在不同 `cardId` 下。

## 七、正则规则汇总（前端静态解析）

所有基础 XML 标签由 `rp-web-xml.js` 的 `parseRPContent` 处理：

| 标签 | 处理方式 | 用途 |
|------|---------|------|
| `<thinking>` | 提取后剥离 → 可折叠展示 | AI 思考过程 |
| `<UpdateVariable>` | **服务端剥离**（前端做最后防线） | 变量更新指令 |
| `<perspective>` | DOMParser 提取 title + html → 可折叠卡片 | 副视角群聊 |
| `<choice>` | 提取文本 → 可点击按钮 | 剧情选项 |
| `<content>` | 提取正文 → Markdown 渲染 | 主体叙事 |

卡片专属的渲染钩子（`<SFW_IMG>` 等）在 Markdown→HTML 之后、DOM 注入之前执行。

## 八、添加新正则钩子

### 方式一：通过角色卡导入（推荐）

在 SillyTavern 中为角色卡添加正则脚本，导出 PNG 后 `node setup.mjs --import`，引擎自动处理。

### 方式二：手动添加

1. 编辑 `.pi/cards/<卡名>/regex_scripts/<卡名>.json`，按 SillyTavern 格式添加条目
2. 在 pi 中 `/card activate <卡名>` → 重启会话
3. 引擎 session_start 时自动重新编译

### 方式三：仅前端钩子

1. 编辑 `.pi/cards/<卡名>/regex_hooks.json`
2. 添加条目：`{ "name": "...", "pattern": "...", "flags": "g", "replacement": "..." }`
3. 刷新浏览器，WebSocket 重连时自动下发
