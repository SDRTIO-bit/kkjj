# 世界书处理机制

> 本文档说明世界书（worldbook）的存储结构、加载机制、搜索方式、大规模条目的处理策略。

---

## 一、核心机制

### 1.1 存储结构

每张卡片独立存储世界书，按 SillyTavern character_book 的 entries 拆分为 `.md` 文件：

```
.pi/cards/<卡名>/worldbook/
└── 角色设定/
    ├── <条目1>.md          # 文件名 = entry.comment 或 entry.name
    ├── <条目2>.md
    ├── <角色名>_角色描述.md  # 卡片全局描述
    └── disabled/           # 禁用条目（不同路线备用）
        └── <条目>.md
```

### 1.2 Token 预算控制

| 参数 | 值 | 说明 |
|------|-----|------|
| 硬上限 | 1500 token | 世界书主动注入的绝对上限（`MAX_WORLDBOOK_TOKENS`） |
| 系统提示索引 | ≤1500 token | `system-prompt.ts` 中按比例截断 |
| 单条目估算 | 条目文本长度 / 1.5（中文）或 / 4（英文） | 粗略估算 |

### 1.3 搜索机制

**`load_worldbook` 工具**：AI 显式调用，按关键词精确搜索。

**主动注入**（`injectRelevantWorldbook`）：每轮 turn_start 从用户消息和已有上下文中提取关键词，匹配世界书文件名。

## 二、关键词提取

### 2.1 动态关键词索引（替代硬编码）

引擎不再硬编码角色名/世界观/地点列表。改为 `session_start` 时从世界书文件**动态构建关键词索引**：

```
session_start 时：
  ├── buildAllCardIndexes()
  │   ├── 遍历所有激活卡片 worldbook/ 目录
  │   ├── 对每个 .md 文件：
  │   │   ├── 提取 yaml front matter 的 keywords 数组
  │   │   ├── 提取 yaml 中的 name 字段
  │   │   └── 按分隔符拆分文件名
  │   ├── 合并去重 → 倒排索引 { 关键词 → [文件列表] }
  │   └── 按卡片 id 隔离存储
  │
  turn_start 时：
    └── extractKeywords() 从所有卡片的索引 key 中匹配用户消息
        不再依赖硬编码的角色名/世界观/地点列表
```

**关键设计**：
- **按卡片隔离**：每张卡独立维护 KeywordIndex，`cardKeywordIndexes` 是 `Map<cardId, KeywordIndex>`
- **关键词来源**：yaml keywords 数组 > yaml name 字段 > 文件名拆分，三者合并去重
- **调用时机**：`index.ts` 的 `session_start` 中，正则钩子加载后立即调用
- **不串卡**：`extractKeywords` 虽然从所有卡片索引收集关键词池，但 `searchWorldbookMultiDir` 按卡片目录隔离搜索

## 三、大规模条目处理策略

当卡片有 500+ 条世界书时，全量注入不可行。当前策略：

### 3.1 按需加载（主要方式）

AI 通过 `load_worldbook` 工具主动查询所需条目：

```
用户: "科尔杜村是什么样的？"
  │
AI 调用: load_worldbook("科尔杜村")
  │
返回: 科尔杜村.md 完整内容
```

**优势**：AI 决定需要什么，精准按需，不浪费 token。

### 3.2 主动注入（辅助方式）

每轮 turn_start 自动匹配当前对话中出现的概念，注入 ≤1500 token 的相关条目。

**去重机制**：`InjectedEntriesTracker` 记录已注入的条目，20 轮后重置，避免重复注入同一设定。

### 3.3 按优先级排序

搜索命中的条目按 `hitCount`（命中关键词数）降序排列，与当前对话最相关的条目优先注入。

### 3.4 禁用条目隔离

原卡中 `enabled: false` 的条目存入 `disabled/` 子目录，不会被自动搜索到。仅当 AI 显式 `load_worldbook(keyword, cardId)` 且指定目录时才能访问。

## 四、文件名规范

### 4.1 SillyTavern 导入的条目

文件名 = `entry.comment || entry.name || "条目" + 序号`。特殊字符被替换为下划线。

### 4.2 搜索匹配规则

`searchWorldbookMultiDir` 的匹配逻辑：

```
1. fileName.includes(keyword)           → 精确包含
2. shortName.includes(normalizedKeyword) → 去除前缀后包含

常见前缀自动剥离：
  "世界观"、"世界观纯爱线"、"归属值"、"情分值"
  "[世界观]"、"[控制器]"、"[档案]"、"[变量]"、"[指南]"
```

### 4.3 搜索目录范围

仅搜索以下子目录（`SEARCH_DIRS` 常量）：

| 目录 | 用途 |
|------|------|
| `世界观/` | 核心设定、规则、机制 |
| `角色设定/` | 角色描述、条目 |
| `身体演化/` | 身体变化描述 |
| `格式指令/` | 输出格式要求 |

不在这些目录下的文件不会被 `load_worldbook` 或主动注入搜索到。

## 五、当前可配置项

| 位置 | 配置项 | 说明 |
|------|--------|------|
| `worldbook.ts: MAX_WORLDBOOK_TOKENS` | 1500 | 主动注入硬上限 |
| `system-prompt.ts: WORLD_BOOK_TOKEN_LIMIT` | 1500 | 系统提示中世界书索引硬上限 |
| `index.ts: userTurnCounter % 20` | 20 轮 | 注入条目去重重置周期 |
| `index.ts: nextUserTurn % 15` | 15 次用户交互 | 强制压缩 + 常开世界书重载 |

## 六、多卡片场景

### 6.1 同名关键词冲突处理

当多张卡同时激活时：
- 同一个关键词可能命中文名相似的条目
- `detectConflicts()` 检测到冲突后标记但**不阻止注入**——两个条目的内容都会被列入候选
- 按 `hitCount` 排序后由 token 预算控制最终注入数量

### 6.2 卡片隔离保证

每张卡的世界书通过 `sourceCard` 标记来源，`load_worldbook` 支持 `cardId` 参数过滤。

## 七、已知问题

| 问题 | 状态 |
|------|------|
| 大卡片（500+ 条目）生成的世界书索引可能超出 token 预算 | ✅ 索引按比例截断 |
| 文件名只支持中文和英文，日文/特殊字符可能匹配失败 | ⚠️ 未处理 |
| 相同条目被重复导入（如多次 `--import` 同一张卡）导致 `_1` 后缀文件 | ⚠️ 需手动清理 |
