# 项目规程

> 本文档是项目的操作规范，涵盖从角色卡导入到引擎运行的全流程约定。

---

## 一、角色卡导入规程

### 1.1 导入命令

```bash
node setup.mjs --import <path-to-card.png>
```

### 1.2 卡片目录结构

```
.pi/cards/<卡名>/
├── worldbook/                  # 世界书（只读）
│   └── 角色设定/
│       ├── <条目>.md
│       └── disabled/
├── state.json                  # ⭐ 只读模板（永不覆盖）
├── config.json                 # 卡片配置
├── APPEND_SYSTEM.md             # 常驻风格规范
├── regex_hooks.json            # 渲染钩子
├── variable_schema.json        # 变量定义
├── regex_scripts/              # 原始正则（备份）
└── tavern_scripts/             # 原始酒馆（备份）
```

### 1.3 通用状态模板

```json
{
  "归属值": 0,
  "情分值": 100,
  "基本信息": { "姓名": "…", "描述": "…" },
  "当前状态": { "所在地点": "未知", "内心想法": "" },
  "_import": { "source": "character_card", … }
}
```

**禁止**在通用模板中加入任何世界观专属字段。

## 二、卡片隔离规程

### 2.1 状态隔离

- 每张卡片独立存储 `state.json`（只读模板）
- 引擎启动时从卡片模板加载初始值
- 运行时动态数据通过 session 历史 `rp-state` 快照持久化
- `saveState()` 不写回卡片目录，保护模板

### 2.2 变量 Schema 隔离

- `variable_schema.json` 按 cardId → charName → {fieldName: type} 三级索引
- 不同卡片的背德值/欲望值互不干扰

### 2.3 正则脚本隔离

- 引擎从各卡片 `regex_scripts/` 目录独立加载编译
- 按 phase 分 prompt（引擎剥离）/ display（前端渲染）

### 2.4 世界书隔离

- `load_worldbook` 支持 `cardId` 参数过滤
- 主动注入按多目录搜索，每条目标记 `sourceCard`

## 三、引擎运行规程

### 3.1 生命周期

```
session_start
  ├── 从卡片模板加载角色初始数据
  ├── 加载正则钩子、动态关键词索引
  ├── 重建 session 状态（如加载历史）
  └── 启动 RP Web 服务器

turn_start
  ├── 世界书主动注入（每 3 次用户交互）
  └── [禁用]格式刷新/状态概览（由 APPEND_SYSTEM 替代）

turn_end
  ├── 用户轮数 +1
  ├── 保存 rp-state 快照到 session
  ├── 每 15 次用户交互：强制压缩
  └── 每 20 次用户交互：重置世界书注入追踪
```

### 3.2 上下文压缩机制

三层协同：

1. **pi 引擎原生压缩**（兜底）：`.pi/settings.json` 中 `"global": 65`，token 达 65% 时自动触发
2. **RP Engine 强制压缩**（主动）：`turn_end` 中 `ctx.compact()`，每 15 次用户交互执行，AI 回复后触发，用户无感知
3. **周期性注入**（辅助）：世界书注入每 3 次用户交互

### 3.3 用户轮数计数

`userTurnCounter` 在 `turn_end` 中递增（只在用户→AI 完整回合结束时触发），所有周期事件基于此判断：

| 事件 | 周期 |
|------|------|
| 强制压缩 | 每 15 次用户交互 |
| 世界书注入 | 每 3 次用户交互 |
| APPEND_SYSTEM 附加 | 每 5 次用户输入 |
| session 清理 | 每 10 次用户交互 |
| 注入去重重置 | 每 20 次用户交互 |

### 3.4 state.json 只读模板机制

**设计原则**：`cards/<卡名>/state.json` 只存储角色初始设定，永不写入动态数据。

- `loadState()`：从卡片模板加载初始值覆盖主 state.json
- `saveStateNow()`：只写主 state.json（快照），不写回卡片目录
- **`/reset` 命令**：从卡片模板文件重建，一键重置所有角色数值
- **历史加载**：从 session 的 `rp-state` 快照重建当时状态

### 3.5 APPEND_SYSTEM.md 前端附加

格式规范不再通过引擎系统提示注入，改为前端每 5 次用户输入拼到消息末尾。

- **位置**：`.pi/extensions/rp-web/rp-web-app.js`
- **原理**：AI 对消息末尾注意力最高
- **前端不可见**：显示时剥离 `[系统风格规范]` 标记后的内容
- **使用**：点击 Web 页面顶栏「📋 规范」按钮开关

### 3.6 AI 工具

| 工具 | 说明 |
|------|------|
| `read_state` | 读取角色状态 |
| `update_state` | 更新归属值/背德值/欲望值/内心想法等 |
| `advance_time` | 推进游戏天数 |
| `load_worldbook` | 按关键词加载世界书 |

### 3.7 用户命令

| 命令 | 功能 |
|------|------|
| `/card list` | 列出所有卡片 |
| `/card activate <id>` | 激活卡片 |
| `/card deactivate <id>` | 休眠卡片 |
| `/reset` | 从卡片模板重置所有角色数值 |
| `/status` | TUI 状态面板 |
| `/history <角色名>` | 变更历史 |
| `/route [路线]` | 选择路线 |
| `/rp` | 帮助 |

## 四、前端渲染管道

```
AI 输出文本
  → rp-web-xml.js: parseRPContent()
  → rp-web-markdown.js: Markdown → HTML
  → rp-web-message-renderer.js: 执行 display 正则钩子
  → 渲染 perspective 折叠卡片 + choice 按钮
  → 注入 DOM
```
